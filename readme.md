## Controllers/: 
    Store controller files that contain business logics and handle HTTP request and response

## Models/: 
    Keep database modle here if your project involves a database. These models defines how data is structured in you application.

## Middleware/: 
    Place middleware functions, which are functions that can process requests before they reach route handles.

## Routes/:
    Define route file to separeate different API endpoint or web routes. This keeps your code midular and easy to maintain.

## index.js:
    This file serves as the entry point for your node.js and express application. It sets up the server and defines the server logic.
